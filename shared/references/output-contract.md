# Output Contract

Use this structure unless the user asks for a different format.

```markdown
## Executive Read

Brief, evidence-backed summary. Include uncertainty if material.

## Facts

- Fact with tool citation.

## Interpretation

- Reasoned implication, clearly separate from facts.

## Watch Items

| Item | Why It Matters | Trigger | Source |
| --- | --- | --- | --- |

## Open Questions

- Missing or unresolved evidence.

## Coverage

- Data available:
- Data missing/stale:
- Fallbacks used:

## Tools Used

| Tool | Purpose | Key Parameters |
| --- | --- | --- |
```

Keep outputs concise for interactive answers. Use tables for monitoring, valuation, catalysts, and portfolio reviews.
